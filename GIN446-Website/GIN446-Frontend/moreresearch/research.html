HTML Patterns

HTML patterns are reusable solutions for common web development problems, including responsive design, form validation, navigation, and media embedding patterns. Examples include:

  • Responsive Patterns: Using meta tags and modern CSS techniques for adaptive designs.
    • <meta name="viewport" content="width=device-width, initial-scale=1.0">
<style>
    body {
        display: flex;
        flex-direction: column;
    }
    @media (min-width: 768px) {
        body {
            flex-direction: row;
             }
    }
</style>
  • Form Validation: Using attributes like 'required', 'pattern', and <datalist> for input suggestions.
    • <form>
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
                      
    <label for="age">Age:</label>
    <input type="text" id="age" name="age" pattern="[0-9]+" title="Only numbers are allowed">
                      
    <label for="browser">Choose a browser:</label>
    <input list="browsers" id="browser" name="browser">
    <datalist id="browsers">
        <option value="Chrome">
        <option value="Firefox">
        <option value="Safari">
        <option value="Edge">
    </datalist>
    <button type="submit">Submit</button>
</form>
  • Navigation Patterns: Semantic navigation using <nav> and structured menus with <ul>.
    • <nav>
    <ul>
        <li><a href="#home">Home</a></li>
        <li><a href="#about">About</a></li>
        <li><a href="#services">Services</a></li>
        <li><a href="#contact">Contact</a></li>
    </ul>
</nav>
  • Media Embedding: Using <picture> for responsive images and <video> for multimedia content.
    • <picture>
    <source srcset="image-large.jpg" media="(min-width: 768px)">
    <source srcset="image-small.jpg" media="(max-width: 767px)">
    <img src="image-default.jpg" alt="Description of image">
</picture>
                    
<video controls>
    <source src="video.mp4" type="video/mp4">
    <source src="video.ogg" type="video/ogg">
                      Your browser does not support the video tag.
</video>

Button vs Input Type="Button"

<button>: More flexible, supports embedding elements like <img> or <span>, and has a default 'type="submit"' unless specified otherwise.

<input type="button">: Simple and restrictive, supports plain text via 'value' but cannot contain nested elements.

The button element is also preferred for accessibility, as it can carry descriptive labels, making it easier for screen readers to interpret.

The input type="button" is faster for scenarios requiring minimal customization, often used in simple scripts or automation.

GET vs POST

Aspect GET POST
Data Visibility Data appended to URL as query strings. Data sent in request body, not visible in URL.
Security Less secure for sensitive data. More secure for sensitive transmissions.
Use Cases Search queries, bookmarkable URLs. Form submissions, file uploads.

Additional Considerations: GET requests can be cached and remain in browser history, while POST requests do not. These differences have implications for usability and user privacy.

PHP

PHP Spaceship Operator

The spaceship operator (<=>) compares two expressions and returns:

  • -1 if the left operand is less than the right.
  • 0 if both operands are equal.
  • 1 if the left operand is greater than the right.
$a = 1 <=> 2; // -1
$b = 2 <=> 2; // 0
$c = 3 <=> 2; // 1

This operator is especially useful in sorting algorithms, such as those utilizing the usort function in PHP. It's concise and avoids the verbosity of standard comparison logic.

PHP Magic Constants

  • __LINE__: Returns the current line number of the file where it is used. This is often used for debugging purposes to identify where a specific line of code is executing.
  • __FILE__: Provides the full path and filename of the file. If used inside an included file, it will return the path of that file. This is particularly useful for file inclusion and dynamic path handling.
  • __DIR__: Returns the directory of the file where it's called. It’s commonly used to locate resources relative to the script’s location. Note that the directory name does not include a trailing slash unless in the root directory.
  • __CLASS__: Outputs the name of the class (case-sensitive) where it is used. When used inside a trait, it returns the class that uses the trait.
  • __METHOD__: Provides the name of the method (case-sensitive) in which it is used. It is often employed in debugging or logging to track the execution flow.
  • __FUNCTION__: Returns the name of the function (case-sensitive) in which it is used. Anonymous functions are given a unique name in the format of `{closure}`.
  • __TRAIT__: Shows the name of the trait where it is used, including the namespace, if any. This is useful for tracking trait usage in larger, modular codebases.
  • __NAMESPACE__: Outputs the name of the current namespace. This is helpful when dealing with namespaced code, ensuring no conflicts arise between global and local namespaces.

Magic constants provide contextual information about where they are called, making them invaluable for debugging, dynamic programming, and implementing modular designs. They are evaluated at compile-time and can be used in a wide range of scenarios, from logging to building reusable components.

XML

External DTDs

System: Specifies an exact location of the DTD file.

<!DOCTYPE note SYSTEM "note.dtd">

Public: Refers to a standardized DTD with an identifier.

<!DOCTYPE note PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

External DTDs provide modularity by allowing reuse across multiple XML documents. However, using external references can introduce validation issues if the file is unavailable.

Standalone Attribute

Indicates whether the XML document relies on external DTDs.

<?xml version="1.0" standalone="yes"?>
  • yes: No external DTD dependencies.
  • no: Relies on external DTDs.

The standalone attribute enhances compatibility in environments where external resources may not be reliably loaded, making XML documents more portable.
